# 1.2 ArkTS 基础语法

> ArkTS 是 HarmonyOS 的应用开发语言，基于 TypeScript 扩展而来，加入了声明式 UI、状态管理等特性。

---

## 一、ArkTS 与 TypeScript 的关系

```
JavaScript
    └── TypeScript（增加了类型系统）
            └── ArkTS（增加了声明式 UI + 装饰器 + 状态管理）
```

如果你熟悉 TypeScript，ArkTS 上手很快；如果没基础，先掌握下面的核心语法即可。

---

## 二、基础语法速览

### 变量声明

```typescript
let name: string = 'HarmonyOS'   // 可变变量
const version: number = 5        // 常量
let flag: boolean = true
let list: string[] = ['a', 'b', 'c']
```

### 函数

```typescript
function add(a: number, b: number): number {
  return a + b
}

// 箭头函数
const multiply = (a: number, b: number): number => a * b
```

### 接口与类型

```typescript
interface User {
  name: string
  age: number
}

const user: User = { name: '张三', age: 20 }
```

### 类

```typescript
class Animal {
  name: string

  constructor(name: string) {
    this.name = name
  }

  speak(): string {
    return `${this.name} 叫了`
  }
}

const cat = new Animal('猫')
console.log(cat.speak())  // 猫 叫了
```

## 三、ArkTS 特有语法

### 文件结构（`.ets` 文件）

每个页面/组件都是一个 `.ets` 文件，固定结构如下：

```typescript
// Index.ets
@Entry          // 标记为页面入口（一个页面只有一个）
@Component      // 标记为组件
struct Index {  // 用 struct 而非 class 定义组件

  build() {     // build() 方法描述 UI，必须实现
    Column() {
      Text('Hello, HarmonyOS 5!')
        .fontSize(28)
        .fontColor('#333333')
    }
    .width('100%')
    .height('100%')
    .justifyContent(FlexAlign.Center)
  }
}
```

> ⚠️ ArkTS 中组件用 `struct` 定义，不是 `class`

---

## 四、装饰器（Decorator）

装饰器是 ArkTS 的核心机制，用 `@` 开头：

| 装饰器 | 作用 |
|--------|------|
| `@Entry` | 标记页面入口组件 |
| `@Component` | 标记自定义组件 |
| `@State` | 组件内部状态，变化时自动刷新 UI |
| `@Prop` | 父→子单向数据传递 |
| `@Link` | 父↔子双向数据绑定 |
| `@Builder` | 标记可复用的 UI 构建函数 |

---

## 五、`@State` 状态管理基础

`@State` 是最常用的装饰器，被它修饰的变量一旦改变，UI 会**自动重新渲染**。

```typescript
@Entry
@Component
struct Counter {
  @State count: number = 0   // 声明状态变量

  build() {
    Column({ space: 20 }) {
      Text(`计数：${this.count}`)  // 使用 this.xxx 引用变量
        .fontSize(32)

      Button('点击 +1')
        .onClick(() => {
          this.count++           // 修改状态 → UI 自动更新
        })
    }
    .width('100%')
    .height('100%')
    .justifyContent(FlexAlign.Center)
  }
}
```

## 六、条件渲染与列表渲染

### 条件渲染 `if / else`

```typescript
build() {
  Column() {
    if (this.isLoggedIn) {
      Text('欢迎回来！')
    } else {
      Text('请先登录')
    }
  }
}
```

### 列表渲染 `ForEach`

```typescript
@State fruits: string[] = ['苹果', '香蕉', '橙子']

build() {
  Column() {
    ForEach(this.fruits, (item: string) => {
      Text(item).fontSize(20)
    })
  }
}
```
